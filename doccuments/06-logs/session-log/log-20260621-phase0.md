## Decisions

1. **Single API project, folder-based separation**: Không split thành multi-project clean architecture trong Phase 0. Dùng folder `Services/`, `Data/`, `Models/`, `Extensions/`, `Middleware/`, `Filters/` để tổ chức code. Có thể refactor thành multi-project sau nếu cần.

2. **Response wrapper dùng `IAsyncResultFilter`**: Filter có access vào compiled `ObjectResult`, tránh phải re-read response stream như middleware. Wrap tự động tất cả `200/201 ObjectResult` vào `ApiResponse<T>` format.

3. **Swagger grouping qua `[ApiExplorerSettings(GroupName)]`**: Mỗi controller khai báo group name, SwaggerUI hiển thị 9 module: Auth, Users, Listings, Interactions, Chat, RentalRequests, Reviews, Notifications, Admin.

4. **JWT dùng `IOptions<T>` pattern**: `JwtSettings` bind từ config, có thể override bằng environment variables.

5. **Connection string chỉ có trong Development**: `appsettings.json` base có empty connection string. Chỉ `appsettings.Development.json` có connection string thật.

6. **Pin .NET SDK 8.0.417 qua `global.json`**: Đảm bảo mọi developer dùng cùng SDK version.

7. **Flutter flavor dùng `--dart-define`**: Không cần package flavor bên ngoài. `String.fromEnvironment('ENV')` đọc config tại runtime.
