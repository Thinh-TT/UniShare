## Decisions

1. **Không dùng ASP.NET Core Identity**: Users là entity thường với `PasswordHash` lưu trực tiếp. JWT service đã có từ Phase 0 dùng custom implementation, không cần Identity scaffolding.

2. **Base entity + ISoftDeletable interface**: `BaseEntity` chứa `Id` (Guid, sinh bằng `Guid.NewGuid()` trong constructor) và `CreatedAt` (DateTime.UtcNow). `ISoftDeletable` interface dành cho 3 entity cần soft delete: Listing, Comment, Message.

3. **IEntityTypeConfiguration per entity**: Mỗi entity có 1 class config riêng trong `Data/Configurations/`. AppDbContext gọi `modelBuilder.ApplyConfigurationsFromAssembly()` để tự động quét.

4. **Enum lưu dạng NVARCHAR string**: Tất cả enum (`ListingType`, `ListingStatus`, `RequestStatus`, `DepositStatus`, `MessageStatus`, `NotificationType`) dùng `.HasConversion<string>()` — human-readable trong DB, khớp database designer.

5. **Delete behavior**: Restrict cho hầu hết FK relationships. Cascade chỉ cho Listing → ListingImages, ListingTags, Upvotes, Comments (child entities chết theo listing). Messages cascade theo Conversation.

6. **Soft delete query filter**: 3 global query filters trên Listing, Comment, Message (`WHERE DeletedAt IS NULL`). Warning `[10622]` từ EF Core về required navigation với query filter là expected behavior — khi listing bị xóa mềm, navigation từ entity con trả về null là đúng.

7. **Seed data với GUID cứng**: Schools (10 đại học Hà Nội), Areas (8 khu vực), Categories (12 danh mục), Tags (15 tags). Dùng GUID format `{prefix}000000-0000-0000-0000-0000000000{seq}` để dễ nhận diện và tránh trùng.

8. **Connection string sync giữa appsettings.json và appsettings.Development.json**: Phát hiện password mismatch khi debug lỗi login failed. Development environment override connection string từ `appsettings.Development.json`, gây sai password `YourDevPassword123!` thay vì `Thinh@12345`.
